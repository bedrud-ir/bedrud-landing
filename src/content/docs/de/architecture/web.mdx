## Authentifizierungsablauf

Der Authentifizierungsstatus wird von einem Zustand-Store in `src/lib/auth.store.ts` verwaltet:

1. **Anmeldung** - Der Benutzer sendet Anmeldeinformationen an `POST /api/auth/login`
2. **Token-Speicherung** - Access- und Refresh-Token werden in `localStorage` gespeichert
3. **Automatische Injektion** - `authFetch` umschlieГџt jeden API-Aufruf und fГјgt den `Authorization: Bearer`-Header hinzu
