Öffnen Sie den Meeting-Link in einem Browser, um beizutreten — keine Installation erforderlich. Für ein natives App-Erlebnis installieren Sie den Client für Ihre Plattform unten.

---

## Browser (keine Installation)

Öffnen Sie die Meeting-URL in **Chrome**, **Firefox**, **Edge** oder **Safari**. Kamera- und Mikrofonzugriff erfordert HTTPS (Ihr Server-Administrator kümmert sich darum).

Keine Downloads, keine Erweiterungen, kein Konto erforderlich, wenn Gastzugriff aktiviert ist.

## Linux-Desktop

<Tabs>
  <TabsList>
    <TabsTrigger value="appimage-recommended">AppImage (empfohlen)</TabsTrigger>
    <TabsTrigger value="flatpak">Flatpak</TabsTrigger>
    <TabsTrigger value="snap">Snap</TabsTrigger>
    <TabsTrigger value="ubuntu-debian-apt">Ubuntu / Debian (apt)</TabsTrigger>
    <TabsTrigger value="arch-linux-aur">Arch Linux (AUR)</TabsTrigger>
    <TabsTrigger value="fedora-rhel-dnf">Fedora / RHEL (DNF)</TabsTrigger>
    <TabsTrigger value="portable-tarxz">Portables tar.xz</TabsTrigger>
  </TabsList>
  <TabsContent value="appimage-recommended">

    Keine Installation erforderlich. Herunterladen, ausführbar machen, starten:

    ```bash
    wget https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-linux-x86_64.AppImage
    chmod +x bedrud-desktop-linux-x86_64.AppImage
    ./bedrud-desktop-linux-x86_64.AppImage
    ```

    <Callout type="tip">
        Um AppImage in Ihren Anwendungsstarter zu integrieren, verwenden Sie [AppImageLauncher](https://github.com/TheAssassin/AppImageLauncher).
    </Callout>
  </TabsContent>
  <TabsContent value="flatpak">

    ```bash
    flatpak install https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-linux-x86_64.flatpak
    flatpak run ir.bedrud.Desktop
    ```
  </TabsContent>
  <TabsContent value="snap">

    ```bash
    sudo snap install bedrud-desktop
    ```
  </TabsContent>
  <TabsContent value="ubuntu-debian-apt">

    Wenn das Bedrud-apt-Repository bereits auf Ihrem System konfiguriert ist:

    ```bash
    sudo apt install bedrud-desktop
    ```

    Siehe die [Paketinstallationsanleitung](/de/docs/guides/packages) für die Repository-Einrichtung.
  </TabsContent>
  <TabsContent value="arch-linux-aur">

    ```bash
    yay -S bedrud-desktop-bin
    ```
  </TabsContent>
  <TabsContent value="fedora-rhel-dnf">

    Wenn das Bedrud-DNF-Repository bereits konfiguriert ist:

    ```bash
    sudo dnf install bedrud-desktop
    ```

    Siehe die [Paketinstallationsanleitung](/de/docs/guides/packages) für die Repository-Einrichtung.
  </TabsContent>
  <TabsContent value="portable-tarxz">

    ```bash
    wget https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-linux-x86_64.tar.xz
    tar xf bedrud-desktop-linux-x86_64.tar.xz
    ./bedrud-desktop
    ```
  </TabsContent>
</Tabs>

    ```bash
    ./bedrud-desktop
    ```
  </TabsContent>
  <TabsContent value="intel-x8664">

    ```bash
    curl -L https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-macos-x86_64.tar.gz | tar xz
    xattr -d com.apple.quarantine bedrud-desktop
    ./bedrud-desktop
    ```
  </TabsContent>
</Tabs>

<Callout type="warning">
    macOS-Builds sind möglicherweise **nicht signiert**. Wenn Gatekeeper die App blockiert, führen Sie den obigen `xattr`-Befehl aus oder gehen Sie zu **Systemeinstellungen → Datenschutz & Sicherheit** und klicken Sie auf **Trotzdem öffnen**.
</Callout>

## Verbindung zu einem Server

Beim ersten Öffnen der Desktop-App werden Sie nach einer Server-URL gefragt.

1. Geben Sie die URL ein, die Ihr Server-Administrator Ihnen bereitgestellt hat (z. B. `https://meet.example.com`)
2. Registrieren Sie ein Konto oder melden Sie sich an
3. Erstellen Sie einen Meetingraum oder treten Sie einem bei

Die Desktop-App unterstützt Verbindungen zu **mehreren Servern**. Weitere können unter **Einstellungen** hinzugefügt werden.
