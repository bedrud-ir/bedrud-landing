Le client de bureau Bedrud est une application native Windows et Linux construite avec **Rust** et le toolkit UI **Slint**. Elle offre la même expérience de réunion principale que les clients web et mobiles, compilée en un seul binaire sans dépendances d'exécution.

## Stack Technologique

| Composant | Technologie |
|-----------|-------------|
| Langage | Rust (stable) |
| Toolkit UI | Slint 1.x |
| Client HTTP | reqwest (async, TLS) |
| Média | LiveKit Rust SDK |
| Stockage | serde_json + keyring OS (libsecret / Windows Credential Store) |
| Système de build | Espace de travail Cargo |

## Prise en Charge des Plateformes

| Plateforme | Rendu | Binaire |
|------------|-------|---------|
| Windows 10/11 | Direct3D 11 | `bedrud-desktop.exe` |
| Linux x86_64 | OpenGL / Vulkan (via EGL/Wayland/X11) | `bedrud-desktop` |
| macOS | _(pas encore - utilisez l'application web)_ | - |

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                   Couche UI Slint                   │
│  (app.slint, login.slint, dashboard.slint, …)       │
│                 Compilée en code natif              │
└────────────────────────┬────────────────────────────┘
                          │  callbacks / liaisons de propriétés
┌────────────────────────┴────────────────────────────┐
│                  ui/bridge.rs                       │
│  Mappe les callbacks Slint → tâches async Rust      │
│  Met à jour les propriétés Slint depuis les threads │
│  d'arrière-plan                                     │
└───────┬──────────────────┬──────────────────┘
        │                          │
┌───────┴──────────┐   ┌───────────┴──────────────────┐
│   module api/    │   │       module livekit/         │
│  (reqwest, JWT)  │   │  (LiveKit Rust SDK, WebRTC)   │
└───────┬──────────┘   └───────────┬──────────────────┘
        │                          │
┌───────┴──────────┐   ┌───────────┴──────────────────┐
│  module store/   │   │  auth/session.rs              │
│  (instances,     │   │  (boucle de rafraîchissement  │
│   settings)      │   │   JWT, stockage keyring)      │
└──────────────────┘   └───────────────────────────────┘
```

### Décisions de conception clés

- **UI à la compilation de Slint** - Les fichiers `.slint` sont compilés en Rust au moment de la construction via `build.rs`. Il n'y a pas de moteur de mise en page à l'exécution ; l'UI est entièrement native.
- **`bridge.rs` comme seule limite UI↔logique** - tous les callbacks Slint sont câblés en un seul endroit, gardant la logique métier hors de la couche UI et rendant le pont facile à auditer.
- **`Weak<AppWindow>` dans les callbacks** - Les poignées UI de Slint sont `!Send`, donc les tâches d'arrière-plan mettent à niveau une référence `Weak` stockée sur le thread UI pour définir les propriétés, plutôt que de partager la poignée entre les threads.
- **Multi-instance via `store/instance.rs`** - identique aux applications mobiles : les instances sont sérialisées dans un fichier JSON dans le répertoire de configuration de l'OS ; chaque instance a son propre `APIClient` et `AuthSession`.

## Construction Locale

### Prérequis

- Chaîne d'outils Rust stable (`rustup toolchain install stable`)
- **Linux :** `libfontconfig`, `libxkbcommon`, `libwayland`, `libgles2`, `libdbus`, `libsecret`

  ```bash
  sudo apt-get install -y \
    libfontconfig1-dev libxkbcommon-dev libxkbcommon-x11-dev \
    libwayland-dev libgles2-mesa-dev libegl1-mesa-dev \
    libdbus-1-dev libsecret-1-dev \
    libasound2-dev
  ```

- **Windows :** Visual Studio Build Tools (MSVC) avec la charge de travail C++

### Build

```bash
# Build de débogage (compilation rapide, pas d'optimisations)
make dev-desktop          # exécute l'application immédiatement après le build

# Build de version
make build-desktop        # → target/release/bedrud-desktop (Linux)
                           # → target/release/bedrud-desktop.exe (Windows)
```

Ou directement avec Cargo :

```bash
cargo build -p bedrud-desktop                          # débogage
cargo build -p bedrud-desktop --release                # optimisé
cargo run   -p bedrud-desktop                          # exécuter immédiatement
```

## CI

L'application de bureau est construite en CI à chaque push vers `main` et sur les pull requests :

| Tâche | Runner | Ce qu'elle vérifie |
|-------|--------|-------------------|
| `Desktop – Build & Test` | `ubuntu-latest` | `cargo build`, `cargo test` |

Les builds de version produisent deux artefacts :

| Artefact | Runner | Format |
|----------|--------|--------|
| `bedrud-desktop-linux-x86_64.tar.xz` | `ubuntu-latest` | tar.xz |
| `bedrud-desktop-windows-x86_64.zip` | `windows-latest` | zip |
