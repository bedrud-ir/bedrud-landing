L'application Android Bedrud est construite avec Jetpack Compose et Kotlin, offrant une expérience de réunion vidéo native avec le mode picture-in-picture, les liens profonds et la prise en charge multi-instance.

## Stack Technologique

| Technologie | Version | Utilisation |
|-------------|---------|-------------|
| Kotlin | 2.1.0 | Langage |
| Jetpack Compose | Material 3 | Toolkit UI |
| Koin | 4.0.0 | Injection de dépendances |
| Retrofit + OkHttp | Dernière | Client HTTP |
| LiveKit Android SDK | 2.23.3 | Média WebRTC |
| Credentials API | Dernière | Prise en charge des clés de passe |
| Encrypted SharedPreferences | Dernière | Stockage sécurisé |
| Coil | Dernière | Chargement d'images |

**Cible :** Min SDK 28, Target SDK 35, JDK 17

## Architecture Multi-Instance

L'application Android prend en charge la connexion simultanée à plusieurs serveurs Bedrud.

```
┌──────────────────────────┐
│     InstanceManager      │  (singleton Koin)
│                          │
│  ┌────────────────────┐  │
│  │   InstanceStore    │  │  Persiste la liste vers SharedPreferences
│  └────────────────────┘  │
│                          │
│  ┌────────────────────┐  │
│  │  Active Instance   │──┼──► InstanceDeps
│  │  (StateFlow)       │  │    ├── AuthManager
│  └────────────────────┘  │    ├── ApiClient
│                          │    ├── AuthApi
│  instances: List<Instance>│    ├── RoomApi
│                          │    ├── PasskeyManager
└──────────────────────────┘    └── RoomManager
```

### Modèle Clé

Toutes les dépendances par instance sont exposées comme `StateFlow<T?>` sur `InstanceManager`. Les composables les collectent :

```kotlin
val authManager = instanceManager.authManager.collectAsState().value ?: return
val roomApi = instanceManager.roomApi.collectAsState().value ?: return
```

Le motif `?: return` garantit que le composable ne s'affiche pas tant que l'instance n'est pas complètement initialisée.

### Flux de Navigation

```
Pas d'instances ──► AddInstanceScreen
Instances existantes, non connecté ──► LoginScreen
Instances existantes, connecté ──► DashboardScreen
```

Le sélecteur d'instance apparaît comme un `ModalBottomSheet` déclenché depuis la barre d'outils du Dashboard.

## Fonctionnalités

### Liens Profonds

L'application gère les URLs correspondant à :

- `https://bedrud.com/m/*` - Rejoindre directement une salle
- `https://bedrud.com/c/*` - Rejoindre une salle par code

Configuré via les filtres d'intent dans `AndroidManifest.xml`.

### Gestion des Appels

- **CallService** - Service de premier plan qui maintient la connexion active pendant les appels
- **CallConnectionService** - S'intègre au framework de téléphonie d'Android pour une interface d'appel appropriée
- Permissions requises : `MANAGE_OWN_CALLS`, `FOREGROUND_SERVICE_PHONE_CALL`, `FOREGROUND_SERVICE_CAMERA`, `FOREGROUND_SERVICE_MICROPHONE`

### Picture-in-Picture

L'écran de réunion prend en charge le mode PiP, permettant aux utilisateurs de voir le flux vidéo tout en utilisant d'autres applications.

### Clés de Passe

Utilise l'API Credentials d'Android pour l'enregistrement et la connexion par clé de passe FIDO2/WebAuthn.

## Construction

```bash
# APK de débogage
make build-android-debug

# APK de version (requiert keystore.properties)
make build-android

# Build + installation sur l'appareil connecté
make release-android

# Ouvrir dans Android Studio
make dev-android
```

### Signature de Version

Les builds de version nécessitent un fichier `keystore.properties` à la racine du projet Android avec votre configuration de signature.
