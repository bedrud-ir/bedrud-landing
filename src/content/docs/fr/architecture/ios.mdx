L'application iOS Bedrud est construite avec SwiftUI, offrant une expérience de réunion vidéo native avec la prise en charge multi-instance et le stockage sécurisé des identifiants.

## Stack Technologique

| Technologie | Version | Utilisation |
|-------------|---------|-------------|
| Swift | 5.9+ | Langage |
| SwiftUI | Dernière | Framework UI |
| LiveKit Swift SDK | 2.0+ | Média WebRTC |
| KeychainAccess | 4.2.2+ | Stockage sécurisé des identifiants |

**Cible de déploiement :** iOS 18.0

## Configuration du Projet

Le projet utilise **XCodeGen** pour la génération du projet à partir de `project.yml` :

- Bundle ID : `com.bedrud.ios`
- Généré avec : `xcodegen generate`

## Architecture Multi-Instance

L'application iOS reflète l'architecture Android pour la prise en charge multi-instance.

```text
┌──────────────────────────┐
│     InstanceManager      │  (ObservableObject)
│                          │
│  @Published properties:  │
│  ├── instances: [Instance]│
│  ├── activeInstance       │
│  ├── authManager          │──► Dépendances
│  ├── apiClient            │    par instance
│  ├── roomManager          │    (container)
│  └── ...                  │
└──────────────────────────┘
```

### Modèle Clé

Les dépendances sont des propriétés `@Published` sur `InstanceManager`, qui est un `ObservableObject`. Les vues le reçoivent via `@EnvironmentObject` :

```swift
struct DashboardView: View {
    @EnvironmentObject var instanceManager: InstanceManager

    var body: some View {
        if let authManager = instanceManager.authManager {
            // Afficher l'UI authentifiée
        }
    }
}
```

### Flux de Navigation

```text
Pas d'instances ──► AddInstanceView
Instances existantes, non connecté ──► LoginView
Instances existantes, connecté ──► DashboardView
```

Le sélecteur d'instance apparaît comme un `.sheet` déclenché depuis la barre d'outils du Dashboard.

## Point d'Entrée de l'Application

`BedrudApp.swift` initialise les services de base et les injecte dans l'environnement SwiftUI :

```swift
@main
struct BedrudApp: App {
    @StateObject var instanceStore = InstanceStore()
    @StateObject var instanceManager = InstanceManager()
    @StateObject var settingsStore = SettingsStore()

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environmentObject(instanceStore)
                .environmentObject(instanceManager)
                .environmentObject(settingsStore)
        }
    }
}
```

## Fonctionnalités

### Stockage Sécurisé

Utilise **KeychainAccess** pour stocker les tokens JWT et les identifiants sensibles, plutôt que UserDefaults.

### Liens Profonds

Gère les URLs pour rejoindre directement des salles et les codes de salle.

### Paramètres

Les préférences utilisateur sont persistantes via `SettingsStore` en utilisant UserDefaults.

## Construction

```bash
# Ouvrir dans Xcode
make dev-ios

# Build d'archive (Version)
make build-ios

# Exporter IPA (requiert ExportOptions.plist)
make export-ios

# Build pour le simulateur (Débogage)
make build-ios-sim
```

### Exigences

- Xcode (dernière version stable)
- Cible de déploiement iOS 18.0
- Pour les builds appareil : compte développeur Apple et profil de provisioning
