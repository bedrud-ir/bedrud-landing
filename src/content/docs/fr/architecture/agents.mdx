Bedrud inclut des agents bot basés sur Python qui peuvent rejoindre des salles de réunion et diffuser du contenu multimédia. Ces bots sont utiles pour la musique de fond, les flux radio ou le partage de contenu vidéo.

## Agents Disponibles

| Agent | Description | Type de média |
|-------|-------------|---------------|
| `music_agent` | Lit des fichiers audio dans une salle | Audio (PCM) |
| `radio_agent` | Diffuse des stations de radio internet | Audio (PCM via FFmpeg) |
| `video_stream_agent` | Partage du contenu vidéo (HLS, MP4) | Vidéo + Audio |

## Fonctionnement des Agents

Tous les agents suivent le même modèle de connexion :

```
1. Analyser l'URL de réunion ──► Extraire le nom de la salle
2. Connexion invité ──► POST /api/auth/guest-login
3. Rejoindre la salle ──► POST /api/room/join (obtenir le token LiveKit)
4. Se connecter à LiveKit ──► Connexion WebSocket
5. Publier les pistes ──► Diffuser les images audio/vidéo
```

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Media Source │────►│  Python Agent│────►│   LiveKit     │
│  (fichier/URL)  │     │  (FFmpeg)    │     │   (WebRTC)    │
└──────────────┘     └──────────────┘     └──────────────┘
                             │
                      Connexion invité + rejoindre
                             │
                      ┌──────────────┐
                      │ Bedrud Server│
                      └──────────────┘
```

## Agent Musique

Lit des fichiers audio (MP3, WAV, etc.) dans une salle de réunion.

### Configuration

```bash
cd agents/music_agent
pip install -r requirements.txt
```

**Dépendances :** `httpx`, `livekit`, `pydub`

### Utilisation

```bash
python agent.py "https://meet.example.com/m/room-name"
```

### Fonctionnement

1. Décode les fichiers audio avec `pydub`
2. Convertit en trames PCM
3. Publie les trames audio vers LiveKit comme piste microphone

> Consultez le [README de l'Agent Musique](https://github.com/bedrud-ir/bedrud/tree/main/agents/music_agent) pour les instructions de configuration et d'utilisation.

## Agent Radio

Diffuse des stations de radio internet dans une salle de réunion en utilisant FFmpeg pour le décodage audio.

### Configuration

```bash
cd agents/radio_agent
pip install -r requirements.txt
```

**Dépendances :** `httpx`, `livekit`

**Configuration système requise :** FFmpeg doit être installé (`brew install ffmpeg` ou `apt install ffmpeg`)

### Utilisation

```bash
python agent.py "https://meet.example.com/m/room-name"
```

### Fonctionnement

1. Se connecte à une URL de flux radio
2. Passe le flux à travers FFmpeg pour décoder en PCM brut
3. Publie les trames audio PCM vers LiveKit

> Consultez le [README de l'Agent Radio](https://github.com/bedrud-ir/bedrud/tree/main/agents/radio_agent) pour les instructions de configuration et d'utilisation.

## Agent Flux Vidéo

Partage la vidéo et l'audio depuis une URL (HLS/m3u8, MP4) dans une salle de réunion.

### Configuration

```bash
cd agents/video_stream_agent
pip install -r requirements.txt
```

**Dépendances :** `httpx`, `livekit`

**Configuration système requise :** FFmpeg doit être installé

### Utilisation

```bash
python agent.py "https://meet.example.com/m/room-name"
```

### Fonctionnement

1. Exécute deux processus FFmpeg en parallèle :
    - **Vidéo :** Décode en trames brutes YUV420p (1280x720 @ 30fps)
    - **Audio :** Décode en échantillons PCM
2. Publie la vidéo comme piste de partage d'écran
3. Publie l'audio comme piste microphone

> Consultez le [README de l'Agent Flux Vidéo](https://github.com/bedrud-ir/bedrud/tree/main/agents/video_stream_agent) pour les instructions de configuration et d'utilisation.

### Spécifications Vidéo

| Paramètre | Valeur |
|-----------|--------|
| Largeur | 1280 |
| Hauteur | 720 |
| FPS | 30 |
| Format de pixel | YUV420p |

## Créer un Agent Personnalisé

Pour créer un nouvel agent, suivez ce modèle :

```python
import httpx
from livekit import rtc

# 1. Analyser l'URL de réunion pour extraire le nom de la salle
room_name = parse_url(meeting_url)

# 2. Connexion invité
client = httpx.Client(base_url=server_url)
resp = client.post("/api/auth/guest-login", json={"name": "Bot Name"})
token = resp.json()["token"]

# 3. Rejoindre la salle
client.headers["Authorization"] = f"Bearer {token}"
resp = client.post("/api/room/join", json={"roomName": room_name})
lk_token = resp.json()["token"]

# 4. Se connecter à LiveKit
room = rtc.Room()
await room.connect(livekit_url, lk_token)

# 5. Publier les pistes
source = rtc.AudioSource(sample_rate=48000, num_channels=1)
track = rtc.LocalAudioTrack.create_audio_track("audio", source)
await room.local_participant.publish_track(track)

# 6. Diffuser les trames
while has_data:
    frame = get_next_frame()
    await source.capture_frame(frame)
```
