این راهنما نصب **سرور بدرود** (API، رابط کاربری وب، سرور مدیا WebRTC) روی یک سرور لینوکسی را پوشش می‌دهد. برای اپلیکیشن‌های کلاینت دسکتاپ و موبایل، به [نصب کلاینت](/docs/getting-started/clients) مراجعه کنید.

---

## پیش‌نیازها

| نیاز | جزئیات |
|------|--------|
| **سیستم‌عامل** | Debian 12+، Ubuntu 22.04+، Arch Linux (rolling)، Fedora 39+ |
| **معماری** | amd64 (x86_64) یا arm64 (aarch64) |
| **دسترسی** | Root یا sudo |
| **سیستم init** | systemd |
| **سخت‌افزار** | حداقل ۱ vCPU، ۵۱۲ MB RAM. برای ۱۰+ اتاق همزمان ۲ vCPU، ۲ GB RAM توصیه می‌شود |
| **پورت‌ها** | `8090` (API + رابط کاربری وب)، `7880` (LiveKit WebSocket)، `50000–60000/udp` (مدیا WebRTC) |
| **TLS خودکار** | همچنین پورت‌های `80` و `443` را باز کنید. نیازمند دامنه با رکورد DNS A که به این سرور اشاره می‌کند |
| **ابزارها** | `curl`، `tar` (در اکثر توزیع‌ها از پیش نصب شده) |

> **دامنه ندارید؟** بدرود با آدرس IP و TLS خودامضا کار می‌کند. مرورگرها هشدار گواهی نشان می‌دهند اما ویدیو و صدا کار می‌کنند. برای گواهی‌های قابل اعتماد در شبکه‌های داخلی، به [راهنمای TLS داخلی](/docs/guides/internal-tls) مراجعه کنید.

## ۲. Docker

بدرود را در یک کانتینر اجرا کنید - بدون نصب سرور لینوکس، بدون systemd، بدون دانلود باینری.

### ایمیج از پیش ساخته‌شده (سریع‌ترین)

```bash
docker pull ghcr.io/bedrud-ir/bedrud:latest

docker run -d --name bedrud \
  -p 8090:8090 \
  -p 7880:7880 \
  -p 50000-60000:50000-60000/udp \
  -v bedrud-data:/var/lib/bedrud \
  ghcr.io/bedrud-ir/bedrud:latest
```

### Docker Compose

```yaml
services:
  bedrud:
    image: ghcr.io/bedrud-ir/bedrud:latest
    ports:
      - "8090:8090"
      - "7880:7880"
      - "50000-60000:50000-60000/udp"
    volumes:
      - bedrud-data:/var/lib/bedrud
    restart: unless-stopped

volumes:
  bedrud-data:
```

### پورت‌ها

| پورت | سرویس | پروتکل |
|------|-------|--------|
| 8090 | API + رابط کاربری وب | HTTP |
| 7880 | LiveKit | WebSocket + HTTP |
| 50000–60000 | مدیا WebRTC | UDP |

### پیکربندی

یک فایل تنظیمات را عبور دهید یا از متغیرهای محیطی استفاده کنید:

```bash
docker run -d --name bedrud \
  -p 8090:8090 \
  -p 7880:7880 \
  -e JWT_SECRET=change-this-to-a-random-string \
  -e LIVEKIT_API_KEY=prodkey \
  -e LIVEKIT_API_SECRET=prodsecret \
  -v bedrud-data:/var/lib/bedrud \
  ghcr.io/bedrud-ir/bedrud:latest
```

<Callout type="warning">
    رمزهای پیش‌فرض JWT و LiveKit فقط برای تست سریع هستند. قبل از در معرض قرار دادن کانتینر به اینترنت، آن‌ها را تغییر دهید.
</Callout>

اسناد کامل Docker شامل حجم‌ها، پروکسی معکوس، و تنظیمات TLS: [راهنمای Docker](/docs/guides/docker).

## ۳. نصب تعاملی

اگر فایل باینری `bedrud` را قبلاً روی سرور دارید:

```bash
sudo ./bedrud install
```

نصب‌کننده IP سرور شما را به طور خودکار تشخیص می‌دهد و برای موارد زیر درخواست می‌کند:

| درخواست | پیش‌فرض | یادداشت |
|---------|---------|---------|
| **آدرس IP** | تشخیص خودکار (اولین IPv4 غیر لوپ‌بک) | برای پذیرش Enter را بزنید، یا IP دیگری تایپ کنید |
| **دامنه** | خالی (فقط IP) | در صورت ارائه با ایمیل، TLS Let's Encrypt را فعال می‌کند |
| **ایمیل** | خالی | برای ثبت‌نام Let's Encrypt استفاده می‌شود (فقط اگر دامنه تنظیم شده باشد) |
| **TLS خودامضا** | بله | هنگامی که دامنه ارائه نشده ارائه می‌شود. پیش‌فرض `Y` است |

همه درخواست‌ها هنگام لوله‌کشی stdin (حالت غیرتعاملی) رد می‌شوند - به جای آن از پرچم‌ها و پیش‌فرض‌ها استفاده می‌شود.

## ۴. غیرتعاملی (مبتنی بر پرچم)

با ارسال پرچم، همه درخواست‌ها را رد کنید.

### Let's Encrypt (دامنه مورد نیاز)

**نیازمند:** رکورد DNS A برای دامنه شما که قبلاً به IP این سرور اشاره می‌کند.

```bash
sudo ./bedrud install --tls --domain meet.example.com --email webmaster@example.com
```

### TLS خودامضا (مبتنی بر IP)

**نیازمند:** چیز اضافی. فوراً کار می‌کند.

```bash
sudo ./bedrud install --tls --ip 1.2.3.4
```

### HTTP ساده (فقط توسعه / داخلی)

**نیازمند:** localhost یا `127.0.0.1`. مرورگرها دوربین/میکروفون را روی غیر HTTPS بجز localhost مسدود می‌کنند.

```bash
sudo ./bedrud install --ip 1.2.3.4
```

### خودامضا برای دامنه (تست قبل از DNS)

**نیازمند:** نام دامنه (DNS نباید بعداً حل شود).

```bash
sudo ./bedrud install --tls --domain meet.example.com
```

### پورت سفارشی

**نیازمند:** پورت که قبلاً در استفاده نیست.

```bash
sudo ./bedrud install --tls --port 8443 --domain meet.example.com
```

### گواهی‌های خارجی

**نیازمند:** فایل‌های fullchain و کلید خصوصی با فرمت PEM.

```bash
sudo ./bedrud install --tls --cert /path/to/fullchain.pem --key /path/to/privkey.pem
```

### پورت‌های LiveKit سفارشی

**نیازمند:** هر سه پورت باز در فایروال شما.

```bash
sudo ./bedrud install --lk-port 9000 --lk-tcp-port 9001 --lk-udp-port 9002
```

### پشت پروکسی معکوس

**نیازمند:** یک پروکسی معکوس (Nginx، Caddy، Traefik، Cloudflare) که TLS را جلوی بدرود خاتمه می‌دهد.

```bash
sudo ./bedrud install --behind-proxy --domain meet.example.com
```

`behindProxy: true` را در تنظیمات تنظیم می‌کند و هدرهای پروکسی قابل اعتماد را پیکربندی می‌کند.

### سرور LiveKit خارجی

**نیازمند:** یک سرور LiveKit جداگانه که قبلاً در حال اجرا است، قابل دسترسی از این ماشین.

```bash
sudo ./bedrud install --external-livekit https://livekit.example.com
```

نصب LiveKit جاسازی‌شده را رد می‌کند. کلید API و رمز باید در هر دو تنظیمات مطابقت داشته باشند.

### دامنه LiveKit جداگانه

**نیازمند:** دامنه‌ای که مستقیماً به این سرور حل می‌شود، از هر CDN عبور می‌کند.

```bash
sudo ./bedrud install --livekit-domain lk.example.com --domain meet.example.com
```

وقتی دامنه اصلی پشت Cloudflare است اما مدیا WebRTC به اتصال مستقیم نیاز دارد مفید است.

### مرجع تمام پرچم‌ها

| پرچم | پیش‌فرض | توضیح |
|------|---------|-------|
| `--tls` | `false` | فعال کردن HTTPS (خودامضا یا Let's Encrypt) |
| `--self-signed` | `false` | تولید گواهی TLS خودامضا |
| `--no-tls` | `false` | غیرفعال کردن TLS به طور کامل (بر `--tls` غلبه می‌کند) |
| `--ip` | تشخیص خودکار | جایگزینی آدرس IP سرور تشخیص داده شده |
| `--domain` | `""` | نام دامنه برای Let's Encrypt یا گواهی خودامضا |
| `--email` | `""` | ایمیل برای ثبت‌نام Let's Encrypt |
| `--port` | `"443"` (TLS) / `"8090"` (HTTP) | جایگزینی پورت شنیدن |
| `--cert` | `""` | مسیر فایل گواهی موجود (PEM) |
| `--key` | `""` | مسیر فایل کلید خصوصی موجود (PEM) |
| `--lk-port` | `"7880"` | پورت API/WebSocket LiveKit |
| `--lk-tcp-port` | `"7881"` | پورت TCP RTC LiveKit |
| `--lk-udp-port` | `"7882"` | پورت UDP RTC LiveKit |
| `--fresh` | `false` | حذف نصب موجود قبل از نصب |
| `--behind-proxy` | `false` | سرور پشت پروکسی معکوس / CDN است |
| `--external-livekit` | `""` | URL یک سرور LiveKit کاملاً خارجی |
| `--livekit-domain` | `""` | دامنه جداگانه برای LiveKit (عبور از CDN) |

## ۵. کاملاً خودکار (CLI Python)

سرور خود را از ماشین محلی خود بسازید، آپلود کنید و پیکربندی کنید.

**نیازمند:** Python 3.12+، [uv](https://docs.astral.sh/uv/) (اجراکننده بسته Python)، دسترسی کلید SSH به سرور هدف، Go 1.24+ و Bun به صورت محلی نصب شده.

```bash
cd tools/cli
uv run python bedrud.py --auto-config \
  --ip <your-server-ip> \
  --user root \
  --auth-key ~/.ssh/id_rsa \
  --domain meet.example.com \
  --acme-email admin@example.com
```

**کاری که انجام می‌دهد:**

۱. باینری بک‌اند (`dist/bedrud`) را می‌سازد
۲. آن را به `bedrud.tar.xz` فشرده می‌کند
۳. آن را به سرور از طریق rsync آپلود می‌کند
۴. سرویس‌های متضاد را پاک می‌کند (Nginx، Apache، Caddy)
۵. فایروال را پیکربندی می‌کند (UFW)
۶. منطق داخلی `install` را روی سرور اجرا می‌کند

| پرچم | توضیح |
|------|-------|
| `--ip` | آدرس IP سرور |
| `--user` | کاربر SSH (پیش‌فرض: root) |
| `--auth-key` | مسیر کلید خصوصی SSH |
| `--domain` | نام دامنه برای Let's Encrypt |
| `--acme-email` | ایمیل برای Let's Encrypt |
| `--uninstall` | حذف بدرود از سرور |

## پس از نصب

<CreateAdmin />

### تأیید سرویس‌ها

```bash
systemctl status bedrud livekit
```

### تأیید API

```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:8090
```

انتظار: `200`

### بررسی فایروال

```bash
sudo ufw status
```

اطمینان حاصل کنید که پورت‌های `8090/tcp`، `7880/tcp`، و `50000:60000/udp` مجاز هستند:

```bash
sudo ufw allow 8090/tcp
sudo ufw allow 7880/tcp
sudo ufw allow 50000:60000/udp
```

### مشاهده لاگ‌ها

```bash
tail -f /var/log/bedrud/bedrud.log
```

یا از طریق journalctl:

```bash
journalctl -u bedrud -u livekit -f
```

## ارتقاء

### مدیر بسته

```bash
# Ubuntu / Debian
sudo apt update && sudo apt upgrade bedrud

# Arch Linux
yay -Syu bedrud-bin

# Fedora / RHEL
sudo dnf upgrade bedrud
```

سرویس‌ها پس از ارتقاء بسته به طور خودکار راه‌اندازی مجدد می‌شوند.

### دانلود باینری

آخرین نسخه را دانلود کنید و نصب را دوباره اجرا کنید. نصب‌کننده امن است که دوباره اجرا شود - باینری و تنظیمات را بازنویسی می‌کند در حالی که پایگاه داده را حفظ می‌کند:

```bash
curl -fsSL -o bedrud.tar.xz https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud_linux_amd64.tar.xz
tar -xJf bedrud.tar.xz && chmod +x bedrud
sudo ./bedrud install --fresh
```

<Callout type="tip">
    `--fresh` نصب قدیمی را اول حذف می‌کند. برای حفظ تنظیمات و پایگاه داده موجود، آن را حذف کنید.
</Callout>

### Docker

```bash
docker pull ghcr.io/bedrud-ir/bedrud:latest
docker stop bedrud && docker rm bedrud
docker run -d --name bedrud \
  -p 8090:8090 \
  -p 7880:7880 \
  -p 50000-60000:50000-60000/udp \
  -v bedrud-data:/var/lib/bedrud \
  ghcr.io/bedrud-ir/bedrud:latest
```

با Docker Compose:

```bash
docker compose pull
docker compose up -d
```

## حذف نصب

برای حذف کامل بدرود:

```bash
sudo bedrud uninstall
```

این سرویس‌ها را متوقف می‌کند، واحدهای systemd را حذف می‌کند، باینری را از `/usr/local/bin` حذف می‌کند، و `/etc/bedrud`، `/var/lib/bedrud`، و `/var/log/bedrud` را پاک می‌کند.

## عیب‌یابی

| مشکل | راه حل |
|------|--------|
| **پورت ۸۰/۴۴۳ در استفاده است** | وب‌سرورهای متضاد را متوقف کنید: `systemctl stop nginx apache2 caddy` |
| **سرویس‌ها شروع نمی‌شوند** | لاگ‌ها را بررسی کنید: `journalctl -u bedrud -u livekit --no-pager -n 50` |
| **مشکلات گواهی TLS** | DNS را تأیید کنید: `dig +short meet.example.com` - باید IP این سرور را برگرداند |
| **تضاد پورت LiveKit** | از پرچم‌های `--lk-port`، `--lk-tcp-port`، `--lk-udp-port` برای تغییر پورت‌ها استفاده کنید |
| **نصب موفقیت‌آمیز بود اما ۴۰۴ در رابط کاربری وب** | Frontend جاسازی نشده است. قبل از `bedrud install` با `make build` بسازید. ساخته‌های فقط باینری frontend را رد می‌کنند |
| **نصب موفقیت‌آمیز بود اما سرویس‌ها خراب شدند** | بررسی شکست‌های بی‌صدا: `journalctl -u bedrud --no-pager -n 100`. تنظیمات را در `/etc/bedrud/config.yaml` تأیید کنید |
| **پورت قبلاً در استفاده است (فرآیند ناشناخته)** | آن را پیدا کنید: `ss -tlnp \| grep 8090` یا `ss -ulnp \| grep 7880` |
| **فایروال مدیا را مسدود می‌کند** | پورت‌ها را باز کنید: `sudo ufw allow 8090/tcp && sudo ufw allow 7880/tcp && sudo ufw allow 50000:60000/udp` |
| **هشدارهای گواهی خودامضا** | مورد انتظار. برای تنظیم CA خصوصی به [راهنمای TLS داخلی](/docs/guides/internal-tls) مراجعه کنید |
| **بدون ویدیو / WebRTC شکست خورد** | برای دسترسی دوربین HTTPS لازم است. از `--tls` استفاده کنید. اطمینان حاصل کنید که UDP ۵۰۰۰۰–۶۰۰۰۰ باز است |
| **پروکسی معکوس: IP کلاینت اشتباه** | از پرچم `--behind-proxy` استفاده کنید. هدرهای پروکسی قابل اعتماد را پیکربندی می‌کند |
