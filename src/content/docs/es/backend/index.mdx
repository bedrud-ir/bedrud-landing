## Introducción
El backend de Bedrud es una plataforma de reuniones de un solo binario escrita en **Go 1.24+**. Todas las dependencias - servidor de medios, frontend web, servidor TURN - están incrustadas en un solo ejecutable.

### Tecnologías Clave
- **Framework Principal:** [Fiber v2](https://gofiber.io/) (Enrutador sin asignación, API estilo Express).
- **Capa de Base de Datos:** [GORM](https://gorm.io/) con soporte para **SQLite** (estándar) y **PostgreSQL** (producción).
- **Motor de Medios:** [LiveKit](https://livekit.io/) (Incrustado como binario hermano).
- **Sistema de Autenticación:** Autenticación multicapa que admite JWT, OAuth2 (Google/GitHub/Twitter) y Passkeys FIDO2 sin contraseña.
- **Activos Estáticos Incrustados:** Utiliza el paquete `embed` de Go 1.16+ para agrupar el frontend de React (activos de cliente pre-renderizados con SSR).
- **Despliegue:** Auto-instalador integrado para Debian/Ubuntu con orquestación de systemd y soporte para ACME (Let's Encrypt).

## ¿Por qué esta Arquitectura?
Las plataformas WebRTC generalmente ejecutan servicios separados para señalización, TURN, servicio web y gestión de base de datos. Bedrud consolida estos en un solo binario mediante:

1.  **Extracción** del binario del servidor de medios en tiempo de ejecución.
2.  **Proxy** del tráfico de medios a través del puerto HTTP(S) principal.
3.  **Automatización** de la configuración a nivel de sistema operativo (SSL, systemd) a través del propio binario.

## Arquitectura de Alto Nivel
El backend sigue una arquitectura en capas estándar:

1.  **Capa de Servidor (`internal/server`):** Configura la aplicación Fiber, rutas y middleware.
2.  **Capa de Manejadores (`internal/handlers`):** Gestiona las solicitudes y respuestas HTTP.
3.  **Capa de Repositorio (`internal/repository`):** Maneja las consultas de la base de datos utilizando GORM.
4.  **Capa de Modelos (`internal/models`):** Define las tablas de la base de datos y las estructuras de Go.
5.  **Capa de Servicio/Lógica:** Integrada en manejadores y repositorios por simplicidad, con paquetes especializados como `internal/auth` y `internal/livekit`.

## Punto de Entrada Principal
La aplicación comienza en `server/cmd/bedrud/main.go`. Tiene tres modos principales:

- `run`: Inicia el servidor completo de reuniones.
- `livekit`: Inicia solo el servidor de medios incrustado.
- `install`: Configura automáticamente el servidor en sistemas Debian/Ubuntu.
