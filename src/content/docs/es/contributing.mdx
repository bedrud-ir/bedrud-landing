Las contribuciones a Bedrud son bienvenidas. Esta guía cubre el proceso para enviar cambios.

> **Nota:** La guía de contribución canónica está en la raíz del repositorio: [CONTRIBUTING.md](https://github.com/bedrud-ir/bedrud/blob/main/CONTRIBUTING.md). Esta página se archiva solo como referencia.

## Primeros pasos

1. Bifurca el repositorio
2. Clona tu bifurcación
3. Crea una rama de características desde `main`
4. Realiza tus cambios
5. Envía una solicitud de extracción

## Requisitos previos

- Go 1.22+
- Bun (para el frontend web)
- Rust (para la aplicación de escritorio)
- Make
- Git

## Configuración del desarrollo

```bash
# Después de bifurcar en GitHub, clona tu bifurcación
git clone https://github.com/<tu-usuario>/bedrud.git
cd bedrud
make init
make dev
```

Consulta el [Flujo de trabajo de desarrollo](/es/docs/guides/development) para instrucciones detalladas de configuración.

## Proceso de solicitud de extracción

1. **Nomenclatura de ramas:** `feature/descripción`, `fix/descripción` o `docs/descripción`
2. **Mensajes de confirmación:** `<acción> <qué> por <por qué>` (por ejemplo, `add user model for auth feature`, `fix login redirect for expired sessions`)
3. **Verificaciones de CI:** Todas las verificaciones de GitHub Actions deben pasar
4. **Descripción:** Incluye qué cambió y por qué

### Verificaciones de CI

Cada PR ejecuta estas verificaciones automáticamente:

| Verificación | Qué valida                              |
|--------------|-----------------------------------------|
| Server       | `go vet`, compilación, pruebas          |
| Web          | Verificación de tipos TypeScript, build |
| Android      | Lint, pruebas unitarias                 |
| iOS          | Compilación, prueba (simulador)         |

## Reportar problemas

Informa problemas en [GitHub Issues](https://github.com/bedrud-ir/bedrud/issues) con:

- Pasos para reproducir
- Comportamiento esperado vs real
- Detalles del entorno (sistema operativo, navegador, versión de la aplicación)

## Documentación

La documentación reside en `docs/` y se construye con [MkDocs Material](https://squidfunk.github.io/mkdocs-material/).

### Vista previa local

```bash
pip install mkdocs-material
mkdocs serve
```

Luego abre `http://localhost:8000` en tu navegador.

### Estructura

Las páginas de documentación se organizan por tema. La navegación se define en `mkdocs.yml` en la raíz del proyecto.
