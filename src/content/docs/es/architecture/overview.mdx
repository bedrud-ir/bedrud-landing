## Componentes

### Servidor (`server/`)

El backend en Go es el nГәcleo de Bedrud. Maneja:

- **REST API** - autenticaciГіn, gestiГіn de salas, operaciones de administraciГіn
- **Servicio de archivos estГЎticos** - el frontend web compilado se incrusta vГӯa `//go:embed`
- **IntegraciГіn con LiveKit** - genera tokens y gestiona salas a travГ©s del LiveKit Protocol SDK
- **Servidor LiveKit incrustado** - el binario del servidor multimedia se ejecuta como proceso hijo

El servidor utiliza el framework web **Fiber** (similar a Express.js en Node.js) y **GORM** como capa ORM. Soporta SQLite para desarrollo y PostgreSQL para producciГіn.

Consulte [Arquitectura del Servidor](/es/docs/architecture/server) para obtener detalles.

### Frontend Web (`apps/web/`)

Una aplicaciГіn **React** construida con TanStack Start, TailwindCSS v4 y shadcn/ui. En producciГіn, se pre-renderiza en el servidor y los activos del cliente se incrustan en el binario Go.

Capacidades clave:

- Interfaz de usuario de videoconferencia con LiveKit Client SDK
- AutenticaciГіn basada en JWT con actualizaciГіn automГЎtica de tokens
- Panel de administraciГіn para gestiГіn de usuarios y salas
- Sistema de diseГұo con biblioteca de componentes consistente

Consulte [Frontend Web](/es/docs/architecture/web) para obtener detalles.

### AplicaciГіn Android (`apps/android/`)

Una aplicaciГіn Android nativa construida con **Jetpack Compose** y **Kotlin**. Usa Koin para inyecciГіn de dependencias y Retrofit para HTTP.

Capacidades clave:

- Experiencia completa de videoconferencia con LiveKit Android SDK
- Modo de imagen en imagen
- Manejo de enlaces profundos (`bedrud.com/m/*` y `bedrud.com/c/*`)
- GestiГіn de llamadas con ConnectionService de Android
- Soporte multi-instancia (conectarse a mГәltiples servidores)

Consulte [AplicaciГіn Android](/es/docs/architecture/android) para obtener detalles.

### AplicaciГіn iOS (`apps/ios/`)

Una aplicaciГіn iOS nativa construida con **SwiftUI**. Usa KeychainAccess para almacenamiento seguro de credenciales y LiveKit Swift SDK para medios.

Capacidades clave:

- Experiencia completa de videoconferencia
- Soporte multi-instancia
- Manejo de enlaces profundos
- Almacenamiento seguro basado en Keychain

Consulte [AplicaciГіn iOS](/es/docs/architecture/ios) para obtener detalles.

### AplicaciГіn de Escritorio (`apps/desktop/`)

Una aplicaciГіn de escritorio nativa para Windows y Linux construida con **Rust** y el toolkit de interfaz de usuario **Slint**. Se compila en un solo binario sin dependencias de tiempo de ejecuciГіn.

Capacidades clave:

- Experiencia completa de videoconferencia vГӯa LiveKit Rust SDK
- Renderizado nativo de Windows (Direct3D 11) y Linux (OpenGL/Vulkan)
- Soporte multi-instancia (conectarse a mГәltiples servidores Bedrud)
- IntegraciГіn con el llavero del sistema operativo para almacenamiento seguro de credenciales

Consulte [AplicaciГіn de Escritorio](/es/docs/architecture/desktop) para obtener detalles.

### Agentes Bot (`agents/`)

Scripts Python que se unen a salas de reuniones como bots y transmiten contenido multimedia:

- **Music Agent** - reproduce archivos de audio
- **Radio Agent** - transmite estaciones de radio de internet
- **Video Stream Agent** - comparte contenido de video (HLS, MP4)

Consulte [Agentes Bot](/es/docs/architecture/agents) para obtener detalles.

## Flujo de ConexiГіn de ReuniГіn

```mermaid
sequenceDiagram
    participant C as Client
    participant S as Bedrud Server
    participant LK as LiveKit SFU

    C->>S: POST /api/room/join
    S->>S: Validate permissions
    S->>C: LiveKit JWT token
    C->>LK: WebSocket connect (with token)
    LK->>C: SDP offer + join response

    Note over C,LK: ICE connectivity check
    C->>LK: Host + STUN + TURN candidates

    alt Direct path (UDP)

    else TURN relay
        C-->>LK: Media via TURN relay

    Note over C,LK: Audio/video tracks flow through SFU
```

1. El cliente solicita unirse a una sala vГӯa la REST API
2. El servidor valida permisos y genera un token LiveKit firmado
3. El cliente se conecta directamente a LiveKit vГӯa WebSocket usando el token
4. ICE recopila candidatos (host, STUN, TURN) y selecciona la mejor ruta
5. Las pistas de audio/video fluyen a travГ©s del SFU de LiveKit

Consulte [Conectividad WebRTC](/es/docs/architecture/webrtc-connectivity) para obtener la pila completa de conectividad.
