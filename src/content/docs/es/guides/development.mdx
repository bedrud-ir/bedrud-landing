Esta guía cubre el flujo de trabajo de desarrollo diario para contribuir a Bedrud.

## Estructura del Repositorio

```
bedrud/
├── server/          # Backend en Go
├── apps/
│   ├── web/         # Frontend en React (TanStack Start)
│   ├── android/     # App de Android
│   ├── ios/         # App de iOS
│   └── desktop/     # App de escritorio (Rust + Slint)
├── agents/          # Agentes de bot en Python
├── packages/        # Tipos TypeScript compartidos
├── tools/cli/       # CLI de despliegue
├── docs/            # Documentación (MkDocs)
├── Cargo.toml       # Raíz del workspace Rust
├── Makefile         # Orquestación de compilación
└── Dockerfile       # Compilación de contenedor
```

## Requisitos Previos

| Herramienta | Requerido Para |
|------|-------------|
| Go 1.24+ | Servidor |
| Bun | Frontend web |
| LiveKit Server | Servidor de medios local |
| Android Studio + JDK 17 | App de Android |
| Xcode | App de iOS |
| Rust (estable) | App de escritorio |
| Python 3.10+ | Agentes de bot |
| FFmpeg | Agentes de radio y video |

## Configuración Inicial

```bash
git clone https://github.com/bedrud-ir/bedrud.git
cd bedrud
make init
```

## Desarrollo Full-Stack

Para ejecutar toda la pila localmente:

```bash
make dev
```

Esto inicia LiveKit, el servidor Go y el servidor de desarrollo de React simultáneamente. Presione `Ctrl+C` para detener todos los procesos.

## Desarrollo del Servidor

El punto de entrada del servidor Go es `server/cmd/server/main.go`.

### Agregar un Endpoint de API

1. **Definir el handler** en `server/internal/handlers/`
2. **Agregar métodos de repositorio** en `server/internal/repository/` si se necesitan nuevas consultas a la base de datos
3. **Registrar la ruta** en la configuración del servidor (generalmente en `main.go` o un archivo de rutas)
4. **Agregar middleware** si el endpoint necesita verificación de autenticación o administrador

### Cambios en la Base de Datos

Agregue o modifique structs de modelos GORM en `server/internal/models/`. GORM migra automáticamente al inicio.

### Documentación de Swagger

Las anotaciones de documentación de API usan el formato swaggo. Después de cambios, regenere:

```bash
cd server
swag init -g cmd/server/main.go
```

> Consulte [Arquitectura del Servidor](/es/docs/architecture/server) para el diseño por capas y la estructura completa de directorios.

## Desarrollo del Frontend Web

El frontend está en `apps/web/` y usa React 19 con TanStack Start.

### Agregar una Página

Cree un nuevo archivo bajo `src/routes/` siguiendo la [convención de enrutamiento basado en archivos de TanStack Router](https://tanstack.com/router/latest/docs/framework/react/guide/file-based-routing) (por ejemplo, `src/routes/settings.tsx`). Exporte una `Route` creada con `createFileRoute`.

### Agregar una Función de Cliente de API

1. Agregue la función en `src/lib/api.ts` usando `authFetch`
2. Defina los tipos TypeScript en línea o en un archivo de tipos compartido
3. Use la función desde su componente de ruta, típicamente a través de un hook `useQuery` de TanStack Query

### Verificación de Tipos

```bash
cd apps/web
bun run check    # ejecuta Biome lint + verificación de tipos TypeScript
```

> Consulte [Arquitectura del Frontend Web](/es/docs/architecture/web) para la estructura completa y patrones del frontend.

## Desarrollo de Android

```bash
make dev-android    # Abre en Android Studio
```

### Patrones Clave

- Todas las pantallas son funciones Composable
- Las dependencias provienen de `InstanceManager` a través de `koinInject()`
- Use `collectAsState().value ?: return` para valores `StateFlow` nulos
- La navegación se maneja en `MainActivity.kt`

### Compilación

```bash
make build-android-debug    # APK de depuración
make build-android          # APK de versión (necesita keystore)
make release-android        # Compilación + instalación en dispositivo
```

> Consulte [Arquitectura de la App de Android](/es/docs/architecture/android) para la arquitectura detallada de la app móvil.

## Desarrollo de iOS

```bash
make dev-ios    # Abre en Xcode
```

### Patrones Clave

- Las vistas son structs de SwiftUI
- Las dependencias provienen de `InstanceManager` a través de `@EnvironmentObject`
- Use enlace opcional para propiedades publicadas nulas
- La navegación usa la pila de navegación nativa de SwiftUI

### Generación del Proyecto

Si modifica `project.yml`, regenere el proyecto de Xcode:

```bash
cd apps/ios
xcodegen generate
```

### Compilación

```bash
make build-ios          # Archivo de versión
make build-ios-sim      # Compilación para simulador
make export-ios         # Exportar IPA
```

> Consulte [Arquitectura de la App de iOS](/es/docs/architecture/ios) para la arquitectura detallada de la app móvil.

## Desarrollo de Escritorio

La app de escritorio está en `apps/desktop/` y es un crate de Rust en el workspace.

### Requisitos Previos (Linux)

```bash
sudo apt-get install -y \
  libfontconfig1-dev libxkbcommon-dev libxkbcommon-x11-dev \
  libwayland-dev libgles2-mesa-dev libegl1-mesa-dev \
  libdbus-1-dev libsecret-1-dev \
  libasound2-dev
```

Windows requiere Visual Studio Build Tools con la carga de trabajo de C++ (MSVC).

### Ejecución

```bash
make dev-desktop    # cargo run -p bedrud-desktop
```

### Compilación

```bash
make build-desktop           # binario optimizado para la plataforma actual
```

### Patrones Clave

- Toda la UI está definida en archivos `.slint` bajo `apps/desktop/ui/`; compilados a Rust en tiempo de compilación por `build.rs`
- `apps/desktop/src/ui/bridge.rs` es el único lugar donde los callbacks de Slint se conectan a la lógica de Rust - mantenga la lógica de negocio fuera de los archivos `.slint`
- Use `Weak<AppWindow>` al generar tareas en segundo plano que necesitan actualizar la UI

> Consulte [Arquitectura de la App de Escritorio](/es/docs/architecture/desktop) para la arquitectura detallada de la app de escritorio.

## Desarrollo de Agentes de Bot

Los agentes están en `agents/` con un directorio por agente.

```bash
cd agents/music_agent
pip install -r requirements.txt
python agent.py "http://localhost:8090/m/test-room"
```

Todos los agentes necesitan un servidor Bedrud en ejecución para autenticarse.

## CI/CD

GitHub Actions se ejecuta en cada envío a `main` y en pull requests:

| Trabajo | Lo que verifica |
|-----|---------------|
| Server | `go vet`, compilación, pruebas |
| Web | Verificación de tipos, compilación |
| Android | Lint, pruebas unitarias, APK de depuración |
| iOS | Compilación + pruebas (simulador, con cobertura) |
| Desktop | `cargo build`, `cargo test` |

Las compilaciones de versión se activan por etiquetas de versión (`v*`).

## Estilo de Código

- **Go:** Formato estándar `gofmt`
- **TypeScript/React:** Biome (configurado en `apps/web/biome.json`)
- **Kotlin:** Formato predeterminado de Android Studio
- **Swift:** Formato predeterminado de Xcode
