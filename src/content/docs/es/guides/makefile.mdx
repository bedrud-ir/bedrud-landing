El `Makefile` raíz es la herramienta central de orquestación de construcción para el monorepo de Bedrud. Proporciona comandos para desarrollo, construcción y despliegue en todas las plataformas.

## Uso

```bash
make <target>
make help       # Mostrar todos los objetivos disponibles
```

## Objetivos de Desarrollo

| Objetivo | Descripción |
|----------|-------------|
| `make init` | Instalar todas las dependencias (web + servidor) |
| `make dev` | Ejecutar LiveKit + servidor + web simultáneamente |
| `make dev-web` | Ejecutar solo el servidor de desarrollo del frontend |
| `make dev-server` | Ejecutar solo el servidor del backend |
| `make dev-livekit` | Ejecutar el servidor LiveKit local |
| `make dev-ios` | Abrir el proyecto iOS en Xcode |
| `make dev-android` | Abrir el proyecto Android en Android Studio |

### `make init`

Instala dependencias tanto para el frontend web como para el servidor de Go:

```bash
cd apps/web && bun install
cd server && go mod tidy && go mod download
```

### `make dev`

Inicia los tres servicios de desarrollo simultáneamente. Presione `Ctrl+C` para detener todos:

- **LiveKit** en `localhost:7880`
- **Servidor** en `localhost:8090`
- **Web** en `localhost:3000`

## Objetivos de Construcción

| Objetivo | Descripción |
|----------|-------------|
| `make build` | Construir frontend + backend (incrustado) |
| `make build-front` | Construir solo el frontend |
| `make build-back` | Construir solo el backend |
| `make build-dist` | Construir tarball de linux/amd64 para producción |

### `make build`

Construcción completa de producción:

1. Ejecuta `bun run build:embed` - pre-renderiza la aplicación React con SSR y copia los activos a `server/frontend/`
2. Construye el binario de Go con el frontend incrustado

Salida: `server/dist/bedrud`

### `make build-dist`

Crea una distribución comprimida lista para producción:

1. Ejecuta `make build`
2. Compila cruzadamente para `linux/amd64` con símbolos de depuración eliminados
3. Crea `dist/bedrud_linux_amd64.tar.xz`

## Objetivos de Android

| Objetivo | Descripción |
|----------|-------------|
| `make build-android-debug` | Construir APK de depuración |
| `make build-android` | Construir APK de lanzamiento (requiere keystore) |
| `make install-android` | Instalar APK de lanzamiento en dispositivo conectado |
| `make release-android` | Construir + instalar APK de lanzamiento |

### Rutas de Salida de APK

- Depuración: `apps/android/app/build/outputs/apk/debug/app-debug.apk`
- Lanzamiento: `apps/android/app/build/outputs/apk/release/app-release.apk`

## Objetivos de iOS

| Objetivo | Descripción |
|----------|-------------|
| `make build-ios` | Construir archivo iOS (Release) |
| `make export-ios` | Exportar IPA del archivo |
| `make build-ios-sim` | Construir para iOS Simulator (Debug) |

### Rutas de Salida

- Archivo: `apps/ios/build/Bedrud.xcarchive`
- IPA: `apps/ios/build/export/Bedrud.ipa`

## Objetivo de Despliegue

| Objetivo | Descripción |
|----------|-------------|
| `make deploy ARGS=...` | Ejecutar la herramienta CLI de despliegue |

### Ejemplo

```bash
make deploy ARGS="--auto-config --ip 1.2.3.4 --user root --domain meet.example.com"
```

Esto ejecuta la CLI de Python en `tools/cli/bedrud.py` con los argumentos proporcionados.

## Objetivos de Limpieza

| Objetivo | Descripción |
|----------|-------------|
| `make clean` | Eliminar artefactos de construcción y binarios compilados |
| `make full-clean` | Eliminar artefactos + todas las dependencias instaladas |

### `make clean`

Elimina:

- `dist/` - tarballs de distribución
- `server/dist/` - binario del servidor compilado
- `apps/web/dist/` - salida de construcción de React
- Contenido de `server/frontend/` - activos del frontend incrustado
- `apps/android/app/build/` - salidas de construcción de Android
- `apps/ios/build/` - salidas de construcción de iOS

### `make full-clean`

Todo lo que hace `clean`, más:

- `apps/web/node_modules/`
- `apps/android/.gradle/`
- Caché de módulos de Go (`go clean -modcache`)

Ejecute `make init` después para reinstalar las dependencias.
