Esta guía cubre todos los métodos de instalación disponibles a través de gestores de paquetes y binarios para Bedrud, tanto el **servidor** como el **cliente de escritorio**.

---

## Instalación del Servidor

### Repositorio apt (Ubuntu / Debian)

La forma más fácil de instalar el servidor Bedrud en sistemas basados en Debian es a través del repositorio apt autoalojado en GitHub Pages.

**1. Agregar el repositorio y la clave GPG:**

```bash
curl -fsSL https://bedrud-ir.github.io/bedrud/bedrud.gpg.key \
  | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/bedrud.gpg

echo "deb https://bedrud-ir.github.io/bedrud stable main" \
  | sudo tee /etc/apt/sources.list.d/bedrud.list
```

**2. Instalar:**

```bash
sudo apt update && sudo apt install bedrud
```

El paquete instala el binario en `/usr/local/bin/bedrud` y registra un servicio systemd. Después de instalar, ejecute el instalador interactivo:

```bash
sudo bedrud install
```

Consulte la [Guía de Instalación](/es/docs/getting-started/installation) para todas las opciones de configuración.

### AUR (Arch Linux)

```bash
yay -S bedrud-bin
```

Después de instalar, ejecute:

```bash
sudo bedrud install
```

---

### Repositorio DNF / RPM (Fedora / RHEL / openSUSE)

**1. Agregar el repositorio:**

```bash
sudo curl -fsSL https://bedrud-ir.github.io/bedrud/dnf/bedrud.gpg.key \
  -o /etc/pki/rpm-gpg/RPM-GPG-KEY-bedrud

sudo curl -fsSL https://bedrud-ir.github.io/bedrud/dnf/bedrud.repo \
  -o /etc/yum.repos.d/bedrud.repo
```

**2. Instalar:**

```bash
sudo dnf install bedrud
```

### Docker

Extraiga la imagen precompilada desde GitHub Container Registry:

```bash
docker pull ghcr.io/bedrud-ir/bedrud:latest
docker run -d \
  --name bedrud \
  -p 8090:8090 \
  -p 7880:7880 \
  -v bedrud-data:/var/lib/bedrud \
  ghcr.io/bedrud-ir/bedrud:latest
```

Consulte la [Guía de Docker](/es/docs/guides/docker) para obtener detalles completos incluyendo montajes de volúmenes y configuración de proxy inverso.

## Instalación del Cliente de Escritorio

### Linux

<Tabs>
  <TabsList>
    <TabsTrigger value="apt">Ubuntu/Debian (apt)</TabsTrigger>
    <TabsTrigger value="aur">Arch Linux (AUR)</TabsTrigger>
    <TabsTrigger value="dnf">Fedora / RHEL (DNF)</TabsTrigger>
    <TabsTrigger value="snap">Snap (cualquier distro)</TabsTrigger>
    <TabsTrigger value="flatpak">Flatpak (cualquier distro)</TabsTrigger>
    <TabsTrigger value="appimage">AppImage (cualquier distro)</TabsTrigger>
    <TabsTrigger value="tarball">tar.xz portátil</TabsTrigger>
  </TabsList>

  <TabsContent value="apt">
    Primero agregue el repositorio apt (consulte [Servidor - Repositorio apt](#repositorio-apt-ubuntu-debian) arriba), luego:

    ```bash
    sudo apt install bedrud-desktop
    ```

    El paquete instala un iniciador `.desktop` y registra la aplicación en su menú de aplicaciones.
  </TabsContent>

  <TabsContent value="aur">
    ```bash
    yay -S bedrud-desktop-bin
    ```
  </TabsContent>

  <TabsContent value="dnf">
    Primero agregue el repositorio DNF (consulte [Servidor - Repositorio DNF](#repositorio-dnf-rpm-fedora-rhel-opensuse) arriba), luego:

    ```bash
    sudo dnf install bedrud-desktop
    ```
  </TabsContent>

  <TabsContent value="snap">
    ```bash
    sudo snap install bedrud-desktop
    ```

    Snap está disponible en cualquier distribución de Linux que soporte el daemon Snap (snapd).
  </TabsContent>

  <TabsContent value="flatpak">
    Instale desde el paquete Flatpak:

    ```bash
    flatpak install https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-linux-x86_64.flatpak
    flatpak run ir.bedrud.Desktop
    ```
  </TabsContent>

  <TabsContent value="appimage">
    El AppImage es autocontenido y funciona en cualquier distribución de Linux sin instalación.

    ```bash
    wget https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-linux-x86_64.AppImage
    chmod +x bedrud-desktop-linux-x86_64.AppImage
    ./bedrud-desktop-linux-x86_64.AppImage
    ```

    <Callout type="tip">
      Para integrar AppImage en su iniciador de aplicaciones, use una herramienta como [AppImageLauncher](https://github.com/TheAssassin/AppImageLauncher).
    </Callout>
  </TabsContent>

  <TabsContent value="tarball">
    Extraiga y ejecute desde cualquier directorio:

    ```bash
    tar xf bedrud-desktop-linux-x86_64.tar.xz
    ./bedrud-desktop
    ```
  </TabsContent>
</Tabs>

### macOS

Están disponibles archivos tar portátiles precompilados tanto para Intel como para Apple Silicon. La aplicación está **sin firmar** - después de extraerla, es posible que deba permitirirla en **Configuración del Sistema → Privacidad y Seguridad**.

<Tabs>
  <TabsList>
    <TabsTrigger value="arm64">Apple Silicon (arm64)</TabsTrigger>
    <TabsTrigger value="x86-64">Intel (x86_64)</TabsTrigger>
    <TabsTrigger value="homebrew">Homebrew Tap</TabsTrigger>
  </TabsList>

  <TabsContent value="arm64">
    ```bash
    curl -L https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-macos-arm64.tar.gz \
      | tar xz
    # Permitir el binario si macOS lo bloquea:
    xattr -d com.apple.quarantine bedrud-desktop
    ./bedrud-desktop
    ```
  </TabsContent>

  <TabsContent value="x86-64">
    ```bash
    curl -L https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud-desktop-macos-x86_64.tar.gz \
      | tar xz
    xattr -d com.apple.quarantine bedrud-desktop
    ./bedrud-desktop
    ```
  </TabsContent>

  <TabsContent value="homebrew">
    ```bash
    brew tap bedrud-ir/bedrud
    brew install bedrud-desktop
    ```
  </TabsContent>
</Tabs>

<Callout type="info">
  Las versiones de macOS están **firmadas con código y notarizadas** cuando `MACOS_CERTIFICATE` está configurado en CI (consulte [Referencia de Secrets](#referencia-de-secrets)). Sin esos secretos, el binario no está firmado - use la solución alternativa `xattr` anterior o haga clic derecho → Abrir en Finder.
</Callout>

<Callout type="warning">
  Advertencia de Gatekeeper para macOS sin firmar: ejecute el comando `xattr` anterior, o abra **Configuración del Sistema → Privacidad y Seguridad** y haga clic en **Abrir de todos modos** después del primer intento de inicio bloqueado.
</Callout>

## Referencia de Secrets

Estos secrets de GitHub Actions desbloquean los canales de distribución opcionales:

| Secret | Canal | Descripción |
|---|---|---|
| `AUR_SSH_PRIVATE_KEY` | AUR | Clave SSH para envío a `aur.archlinux.org` |
| `APT_GPG_PRIVATE_KEY` | repositorio apt | Clave GPG para firmar el archivo `Release` |
| `APT_GPG_KEY_ID` | repositorio apt | ID de clave de lo anterior |
| `RPM_GPG_PRIVATE_KEY` | repositorio DNF | Clave GPG para firmar paquetes RPM |
| `SNAPCRAFT_STORE_CREDENTIALS` | Snap Store | Inicio de sesión desde `snapcraft export-login` |
| `CHOCOLATEY_API_KEY` | Chocolatey | Clave API desde chocolatey.org |
| `HOMEBREW_TAP_TOKEN` | tap Homebrew | PAT de GitHub con acceso de escritura a `bedrud-ir/homebrew-bedrud` |
| `WINGET_GITHUB_TOKEN` | WinGet | PAT de GitHub para enviar PR a `microsoft/winget-pkgs` |
| `MACOS_CERTIFICATE` | firma macOS | Certificado `.p12` de ID de desarrollador codificado en base64 |
| `MACOS_CERTIFICATE_PWD` | firma macOS | Contraseña para el archivo `.p12` |
| `MACOS_TEAM_ID` | firma/notarización macOS | ID de equipo de desarrollador de Apple |
| `MACOS_APPLE_ID` | notarización macOS | Dirección de correo electrónico de Apple ID |
| `MACOS_APP_PWD` | notarización macOS | Contraseña específica de la aplicación para notarytool |
| `IOS_P12_BASE64` | IPA iOS | Certificado de distribución iOS |
| `TELEGRAM_BOT_TOKEN` | Telegram | Token de bot para notificaciones de lanzamiento |
| `TELEGRAM_CHAT_ID` | Telegram | ID de canal/grupo objetivo |

Todos los secrets son opcionales - si están ausentes, el trabajo de ese canal se omite silenciosamente.
