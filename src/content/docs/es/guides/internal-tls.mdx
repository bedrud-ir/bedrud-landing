Para redes aisladas - sin dominio público, sin internet saliente, conectividad restringida.

Los navegadores requieren HTTPS confiable para cámara/micrófono vía WebRTC. Los certificados autofirmados funcionan pero muestran advertencias. Para una configuración limpia, genere una CA privada y distribúyala a los clientes.

## Generar CA Privada y Certificado del Servidor

```bash
openssl genrsa -out ca.key 4096
openssl req -new -x509 -days 3650 -key ca.key -out ca.crt \
  -subj "/CN=Bedrud Internal CA"

openssl genrsa -out server.key 2048
openssl req -new -key server.key -out server.csr \
  -subj "/CN=<su-ip-o-hostname-del-servidor>"
openssl x509 -req -days 365 -in server.csr \
  -CA ca.crt -CAkey ca.key -CAcreateserial -out server.crt

sudo ./bedrud install --tls --cert server.crt --key server.key --ip SU_IP_DEL_SERVIDOR
```

## Agregar CA a Almacenes de Confianza del Cliente

Distribuya `ca.crt` a todas las máquinas cliente:

### Windows

```powershell
certmgr.msc  # Clic derecho en ca.crt → Instalar Certificado → CAs Raíz de Confianza
```

### macOS

```bash
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain ca.crt
```

### Linux (Debian/Ubuntu)

```bash
sudo cp ca.crt /usr/local/share/ca-certificates/bedrud-ca.crt
sudo update-ca-certificates
```

### Linux (Arch/Fedora)

```bash
sudo cp ca.crt /etc/pki/ca-trust/source/anchors/bedrud-ca.crt
sudo update-ca-trust
```

Reinicie los navegadores, luego abra `https://SU_IP_DEL_SERVIDOR` (reemplace con la dirección IP real del servidor, por ejemplo, `192.168.1.100`). Después de reiniciar, el certificado será confiable y el acceso a cámara/micrófono funcionará sin advertencias.

## Solución de Problemas

| Problema | Solución |
|---------|-----|
| El navegador aún advierte certificado no confiable | Limpiar el caché del navegador, reiniciar el navegador después de instalar la CA |
| Cámara/micrófono bloqueado | Verifique que la CA esté en CAs Raíz de Confianza, no intermedios |
| Certificado expirado | Regenerar con un valor `-days` más largo |
| Desajuste de CN | Asegúrese de que el CN del certificado del servidor coincida con la IP o el hostname en la URL |
