Bedrud включает ботов-агентов на Python, которые могут подключаться к комнатам и транслировать медиаконтент. Они полезны для фоновой музыки, радиопотоков или демонстрации видеоконтеинта.

## Доступные агенты

| Агент | Описание | Тип медиа |
|-------|-------------|-----------|
| `music_agent` | Воспроизводит аудиофайлы в комнате | Аудио (PCM) |
| `radio_agent` | Транслирует интернет-радиостанции | Аудио (PCM через FFmpeg) |
| `video_stream_agent` | Демонстрирует видеоконтент (HLS, MP4) | Видео + Аудио |

## Как работают агенты

Все агенты следуют одному паттерну подключения:

```
1. Parse meeting URL ──► Extract room name
2. Guest login ──► POST /api/auth/guest-login
3. Join room ──► POST /api/room/join (get LiveKit token)
4. Connect to LiveKit ──► WebSocket connection
5. Publish tracks ──► Stream audio/video frames
```

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Media Source │────►│  Python Agent│────►│   LiveKit     │
│  (file/URL)  │     │  (FFmpeg)    │     │   (WebRTC)    │
└──────────────┘     └──────────────┘     └──────────────┘
                            │
                     Guest login + join
                            │
                     ┌──────────────┐
                     │ Bedrud Server│
                     └──────────────┘
```

## Музыкальный агент

Воспроизводит аудиофайлы (MP3, WAV и др.) в комнате конференции.

### Установка

```bash
cd agents/music_agent
pip install -r requirements.txt
```

**Зависимости:** `httpx`, `livekit`, `pydub`

### Использование

```bash
python agent.py "https://meet.example.com/m/room-name"
```

### Принцип работы

1. Декодирует аудиофайлы с помощью `pydub`
2. Конвертирует в PCM-фреймы
3. Публикует аудиофреймы в LiveKit как дорожку микрофона

> См. [README музыкального агента](https://github.com/bedrud-ir/bedrud/tree/main/agents/music_agent) для инструкций по установке и использованию.

## Радио-агент

Транслирует интернет-радиостанции в комнату конференции, используя FFmpeg для декодирования аудио.

### Установка

```bash
cd agents/radio_agent
pip install -r requirements.txt
```

**Зависимости:** `httpx`, `livekit`

**Системное требование:** должен быть установлен FFmpeg (`brew install ffmpeg` или `apt install ffmpeg`)

### Использование

```bash
python agent.py "https://meet.example.com/m/room-name"
```

### Принцип работы

1. Подключается к URL радиопотока
2. Передаёт поток через FFmpeg для декодирования в Raw PCM
3. Публикует PCM-аудиофреймы в LiveKit

> См. [README радио-агента](https://github.com/bedrud-ir/bedrud/tree/main/agents/radio_agent) для инструкций по установке и использованию.

## Видеостриминг-агент

Демонстрирует видео и аудио по URL (HLS/m3u8, MP4) в комнате конференции.

### Установка

```bash
cd agents/video_stream_agent
pip install -r requirements.txt
```

**Зависимости:** `httpx`, `livekit`

**Системное требование:** должен быть установлен FFmpeg

### Использование

```bash
python agent.py "https://meet.example.com/m/room-name"
```

### Принцип работы

1. Запускает два процесса FFmpeg параллельно:
    - **Видео:** декодирует в Raw-фреймы YUV420p (1280x720 @ 30fps)
    - **Аудио:** декодирует в PCM-сэмплы
2. Публикует видео как дорожку демонстрации экрана
3. Публикует аудио как дорожку микрофона

> См. [README видеостриминг-агента](https://github.com/bedrud-ir/bedrud/tree/main/agents/video_stream_agent) для инструкций по установке и использованию.

### Характеристики видео

| Параметр | Значение |
|---------|-------|
| Ширина | 1280 |
| Высота | 720 |
| FPS | 30 |
| Формат пикселей | YUV420p |

## Создание собственного агента

Для создания нового агента следуйте этому паттерну:

```python
import httpx
from livekit import rtc

# 1. Parse the meeting URL to extract room name
room_name = parse_url(meeting_url)

# 2. Guest login
client = httpx.Client(base_url=server_url)
resp = client.post("/api/auth/guest-login", json={"name": "Bot Name"})
token = resp.json()["token"]

# 3. Join room
client.headers["Authorization"] = f"Bearer {token}"
resp = client.post("/api/room/join", json={"roomName": room_name})
lk_token = resp.json()["token"]

# 4. Connect to LiveKit
room = rtc.Room()
await room.connect(livekit_url, lk_token)

# 5. Publish tracks
source = rtc.AudioSource(sample_rate=48000, num_channels=1)
track = rtc.LocalAudioTrack.create_audio_track("audio", source)
await room.local_participant.publish_track(track)

# 6. Stream frames
while has_data:
    frame = get_next_frame()
    await source.capture_frame(frame)
```
