Разверните Bedrud и присоединитесь к видеоконференции за 5 минут.

**В этом руководстве:** Скачать → Установить → Проверить → Подключиться. Альтернативные способы см. в разделе [Установка](/ru/docs/getting-started/installation). Настройку параметров см. в [Конфигурация](/ru/docs/getting-started/configuration).

> **Единый бинарный файл, после загрузки интернет не нужен.** Включает встроенный медиасервер и SQLite — работает в изолированных сетях. Скопируйте файл, выполните одну команду — готово.

---

## Что вам понадобится

| Требование | Подробности |
|-------------|---------|
| Linux-сервер | amd64, с доступом `sudo` и `systemd`. Windows: Docker или WSL2. macOS: сборка из исходного кода |
| Открытые порты | `8090`, `7880`, `50000–60000/udp`. Для авто-TLS: также `80` и `443` |
| Инструменты | `curl`, `tar` |
| Домен (опционально) | С DNS A-записью, указывающей на ваш сервер. Нет домена? Работает с IP — см. альтернативы ниже |

> **Как это работает:** Вы запускаете один **сервер** Bedrud. Люди подключаются к конференциям в своём **браузере** (без установки) или через **нативные приложения** (Android, Windows, macOS, Linux).

---

## 1. Загрузка

```bash
curl -fsSL -o bedrud.tar.xz https://github.com/bedrud-ir/bedrud/releases/latest/download/bedrud_linux_amd64.tar.xz
tar -xJf bedrud.tar.xz
chmod +x bedrud
```

> **Нет интернета на сервере?** Загрузите на любой компьютер, перенесите через USB или `scp bedrud.tar.xz user@server:/tmp/`, затем распакуйте. Бинарный файл полностью автономен — никаких исходящих запросов после переноса.

> **Загрузка не удалась?** Проверьте доступ в интернет или убедитесь, что URL доступен: [github.com/bedrud-ir/bedrud/releases](https://github.com/bedrud-ir/bedrud/releases/latest).

---

## 2. Установка

Замените `meet.example.com` и `admin@example.com` на свои значения:

```bash
sudo ./bedrud install --tls --domain meet.example.com --email admin@example.com
```

Устанавливает бинарный файл, генерирует конфигурацию, создаёт службы systemd, подготавливает TLS и запускает всё.

> **Порт 80 или 443 занят?** Остановите конфликтующие службы: `sudo systemctl stop nginx apache2 caddy`, затем запустите установку снова.

> **Нет домена?** Используйте IP-адрес сервера:
> ```bash
> sudo ./bedrud install --tls --ip 1.2.3.4
> ```
> Будет использован самоподписанный сертификат. Браузер покажет предупреждение, но видео будет работать. Для доверенных сертификатов во внутренних сетях см. [Руководство по внутреннему TLS](/ru/docs/guides/internal-tls).

---

## 3. Проверка

```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:8090
```

Ожидаемый результат: `200`

> **Не 200?** Проверьте статус служб: `sudo systemctl status bedrud livekit`. Проверьте журналы: `journalctl -u bedrud -u livekit --no-pager -n 50`.

---

## 4. Регистрация и создание администратора

Откройте `https://meet.example.com` (или `https://<ваш-ip>:8090`) в браузере:

<CreateAdmin createOption />

3. Войдите в систему (или обновите страницу) — в боковой панели появится раздел **Администрирование** (Обзор, Пользователи, Комнаты, Настройки). Готово — вы администратор.
4. **Создайте комнату** → **Подключитесь** → **Включите видео/аудио** — разрешите доступ к камере и микрофону при запросе

> **Панель администратора** — После назначения роли войдите в систему для доступа к панели администратора по адресу `/dashboard/admin`. Управляйте пользователями, комнатами, системными настройками и токенами приглашений.

> **Справка по CLI** — Полное управление пользователями: `bedrud user promote`, `demote`, `create`, `delete`.

**Нативные клиенты** для Android, Windows, macOS и Linux: [GitHub Releases](https://github.com/bedrud-ir/bedrud/releases/latest). Или поделитесь ссылкой на конференцию — участники подключатся в любом браузере.

## Альтернативные способы установки

### Docker

```bash
docker run -d --name bedrud \
  -p 8090:8090 \
  -p 7880:7880 \
  -p 50000-60000:50000-60000/udp \
  -v bedrud-data:/var/lib/bedrud \
  ghcr.io/bedrud-ir/bedrud:latest
```

Пользовательская конфигурация, Docker Compose и переменные окружения: см. [Руководство по Docker](/ru/docs/guides/docker).

### Сборка из исходного кода

Требуется [Go 1.24+](https://go.dev/dl/) и [Bun](https://bun.sh/):

```bash
git clone https://github.com/bedrud-ir/bedrud.git
cd bedrud
make init
make local-run
```

`make init` устанавливает LiveKit, создаёт конфигурацию, загружает зависимости. `make local-run` собирает и запускает на `http://localhost:8090`.

### Обычный HTTP (только для разработки / localhost)

```bash
sudo ./bedrud install --ip 127.0.0.1
```

> **Предупреждение:** Браузеры блокируют доступ к камере и микрофону по не-HTTPS, кроме `localhost`. HTTP — только для разработки.

### Системы, отличные от Linux

Команда `install` работает только в Linux. Разместите бинарный файл в PATH вручную:

| ОС      | Способ                                       |
|---------|-------------------------------------------|
| macOS   | `sudo cp bedrud /usr/local/bin/`          |
| Windows | Переместите `bedrud.exe` в папку из `%PATH%` |

## Конфигурация

Установщик генерирует `/etc/bedrud/config.yaml`. Значения по умолчанию подходят для большинства конфигураций.
Измените `jwtSecret` и `sessionSecret` для production, затем перезапустите:

```bash
sudo systemctl restart bedrud livekit
```

Для настройки вместимости комнат, таймаутов или медиапортов см. [Раздел LiveKit в Конфигурации](/ru/docs/getting-started/configuration#livekit-configuration).

## Устранение неполадок

| Проблема | Решение |
|---------|-----|
| Порт 80/443 занят | `sudo systemctl stop nginx apache2 caddy` |
| Службы не запускаются | `journalctl -u bedrud -u livekit --no-pager -n 50` |
| Ошибки сертификата TLS | Проверьте DNS: `dig meet.example.com`. Должен указывать на IP сервера |
| Нет видео / ошибка WebRTC | Для камеры требуется HTTPS. Используйте `--tls`. Откройте UDP 50000–60000 на файрволе |
| Конфликт портов LiveKit | Используйте флаги `--lk-port`, `--lk-tcp-port`, `--lk-udp-port` |
| Файрвол блокирует медиа | `sudo ufw allow 8090/tcp && sudo ufw allow 7880/tcp && sudo ufw allow 50000:60000/udp` |
| Предупреждения самоподписанного сертификата | См. [Руководство по внутреннему TLS](/ru/docs/guides/internal-tls) для настройки приватного CA |

## Удаление

```bash
# Установка из бинарного файла
sudo ./bedrud uninstall

# Docker
docker stop bedrud && docker rm bedrud && docker volume rm bedrud-data

# Сборка из исходного кода
# Ctrl+C для остановки, затем: make clean
```
